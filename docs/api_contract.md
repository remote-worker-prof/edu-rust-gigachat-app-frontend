# Контракт API

## Базовый URL

Указывается через переменную окружения API_BASE_URL.

## Эндпоинты

1. GET /
   - Назначение: текстовое описание API.
   - Ответ: текст.

2. GET /health
   - Назначение: состояние сервиса.
   - Ответ: JSON со статусом, версией и флагом gigachat.

3. POST /ask
   - Назначение: отправка вопроса и получение ответа.
   - Тело запроса: JSON с полем question.
   - Ответ: JSON с полями answer, source и system_prompt_applied.

## Ошибки

- Неверный JSON: HTTP 400.
- Пустой вопрос: JSON с полем error.
- Непредусмотренные ошибки: HTTP 500.
