# Требования к UI/UX (учебная версия)

Документ описывает, каким должен быть интерфейс с точки зрения удобства и
обучаемости. Формулировки ориентированы на студентов без опыта в дизайне.

## 1. Общая цель

Интерфейс должен быть:
- простым и понятным;
- не перегруженным деталями;
- ориентированным на обучение, а не на «красивость любой ценой».

## 2. Основные экраны

### 2.1. Главный экран

Минимальный набор элементов:
- поле ввода вопроса;
- кнопка отправки;
- область, где появляется ответ.

### 2.2. Экран состояния

- информация о доступности API;
- индикация режима (mock / gigachat).
- поле для базового URL API (чтобы менять адрес без пересборки).

## 3. Поведение интерфейса

Обязательные сценарии:

- **Индикация ожидания**: пользователь должен видеть, что запрос в работе.
- **Ошибки сети**: сообщение должно быть кратким и понятным.
- **Пустой вопрос**: отправка должна быть запрещена.

## 4. Доступность (простыми словами)

**Доступность** — это возможность пользоваться интерфейсом без
дополнительных усилий. Даже если человек плохо видит или устал.

Минимальные требования:
- хороший контраст между текстом и фоном;
- нельзя полагаться только на цвет (например, «красный = ошибка»); нужно
  ещё и текстовое пояснение;
- читаемый размер шрифта.

## 5. Локализация

- основной язык интерфейса — русский;
- ответы приходят от API и отображаются без изменений.
