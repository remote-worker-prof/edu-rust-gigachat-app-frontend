# Учебные материалы к лабораторной работе (UI)

Данная директория содержит полный комплект учебных материалов для
лабораторной работы по разработке web‑интерфейса на Rust (Yew), который
работает как клиент к API проекта `rust-gigachat-app`.

## Содержание

1. **lab_work.md** — текст лабораторной работы с целями, теорией и шагами.
2. **lab_02_web_interface_introduction.md** — вводный материал для Moodle.
3. **group_work_guide.md** — методические указания по групповой работе.
4. **library_reference.md** — справочные материалы по библиотекам UI‑проекта.
5. **resources_collection.md** — подборка ссылок на официальные источники.
6. **git_github_setup.md** — практикум по SSH‑ключам и подключению к GitHub.
7. **git_version_control.md** — практикум по базовым операциям Git.
8. **beads_guide.md** — практикум по issue‑трекеру beads и workflow с Git.

## Для преподавателя

Рекомендуется начинать с файла `lab_work.md`. Он содержит полный сценарий
лабораторной работы и необходимый минимум теории.

## Для студентов

1. Ознакомьтесь с основной документацией проекта:
   - `docs/build_and_run.md` — запуск и сборка UI;
   - `docs/api_contract.md` — контракт API и формат запросов;
   - `docs/common_issues.md` — частые проблемы и их причины;
   - `docs/yew_ui_guide.md` — как устроены компоненты в Yew;
   - `docs/ui_ux_requirements.md` — требования к интерфейсу;
   - `docs/stack_guide.md` — обзор используемого стека.
2. Прочитайте и выполните шаги лабораторной работы (`lab_work.md`).
3. При работе с GitHub используйте практикум `git_github_setup.md`.
4. При работе с Git используйте практикум `git_version_control.md`.
5. При работе с задачами используйте практикум `beads_guide.md`.

## Связанные документы

- `docs/common_issues.md` — типовые ошибки (включая CORS).
- `docs/build_and_run.md` — запуск двух серверов (backend + UI).
- `lab_work.md` — подробный разбор CORS и способов его устранения.
- `agents-issue-workflow.cncf.yaml` — формальное описание рабочего процесса (служебно, не обязательно для выполнения ЛР).
