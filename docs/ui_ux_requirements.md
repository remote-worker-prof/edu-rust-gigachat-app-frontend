# Требования к UI/UX

## Общая цель

Интерфейс должен быть простым, понятным и ориентированным на обучение.
Дизайн не должен перегружать пользователя и отвлекать от содержания.

## Основные экраны

1. Главный экран
   - Ввод вопроса.
   - Кнопка отправки.
   - Область ответа.

2. Экран состояния
   - Информация о доступности API.
   - Индикация режима (mock / gigachat).

## Поведение

- Явная индикация ожидания ответа.
- Обработка сетевых ошибок с понятным сообщением.
- Запрет отправки пустых вопросов.

## Доступность

- Контрастность текста и фона.
- Минимальная зависимость от цвета.
- Читабельные размеры шрифтов.

## Локализация

- Базовый язык интерфейса: русский.
- Текст ответов поступает от API.
