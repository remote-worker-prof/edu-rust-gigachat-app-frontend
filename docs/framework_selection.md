# Выбор фреймворка Web UI на Rust

## Критерии

- Зрелость и стабильность экосистемы.
- Документация и количество обучающих материалов.
- Сообщество и длительность развития.
- Пригодность для учебного проекта и простота входа.

## Итоговый выбор

**Yew** выбран как основной фреймворк для данного проекта.

### Обоснование

- Длительная история развития и активная поддержка.
- Стабильные релизы, ориентированные на продуктивные Web UI.
- Понятная модель компонентов и маршрутизации.
- Подходит для учебных целей благодаря предсказуемой архитектуре.

## Альтернативы

- **Dioxus**: активно развивается и предлагает единый подход для разных платформ.
- **Leptos**: современный full-stack подход, удобен для SSR, но быстрее меняется.

## Примечание

Альтернативы можно рассматривать в продвинутых заданиях, но базовый курс
ориентирован на Yew для снижения сложности и повышения воспроизводимости.
